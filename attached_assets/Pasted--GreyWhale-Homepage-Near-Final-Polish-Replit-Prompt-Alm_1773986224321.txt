## 7. FLOATING BACKGROUND ELEMENTS — FILL THE WHITE SPACE

The left and right margins of the page have too much empty white space. Add floating decorative elements in these margins that create visual interest and reinforce the tech/code aesthetic.

**What to add:**

Scattered, semi-transparent code snippets and terminal fragments floating in the margins. They should be:

- Very low opacity (8-12%) so they don't compete with content
- Different sizes — some small fragments, some longer blocks
- Slowly floating/drifting with subtle CSS animation — gentle vertical movement (translateY oscillating 10-20px over 10-15 seconds), maybe slight rotation
- Fixed position or parallax-scroll so they move at a slightly different speed than the content
- Content should look like real code fragments:
  - `<meta name="description" content="..."/>`
  - `schema: LocalBusiness`
  - `lighthouse: 98`
  - `deploy: production`
  - `mesh.connect(node_15)`
  - `sitemap.xml generated`
  - `route: /services`
  - `SEO: optimized`
  - HTML tag fragments like `<section>`, `</div>`, `className="hero"`

- Use a monospace font at small sizes (11-13px)
- Mix of light gray text on white background sections and slightly lighter on the tinted sections
- They should appear on DESKTOP ONLY — hide completely on mobile/tablet
- Position them in the gutters — the space between the page content and the browser edge
- They should NOT overlap with or interfere with any actual content
- Think of it like looking at the page and seeing the "source code" of the site bleeding through the edges

This creates the feeling that the whole site is built on real technology — the code is literally visible in the background. It's a subtle flex that reinforces the technical credibility without saying a word.
