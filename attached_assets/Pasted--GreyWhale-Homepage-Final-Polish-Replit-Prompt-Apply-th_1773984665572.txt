The page currently forces the user to snap back to the top if they scroll down before the terminal typing animation finishes. This is breaking the entire user experience. FIX THIS IMMEDIATELY.

The terminal animation should NOT block or interfere with scrolling in any way. The user must be able to scroll freely at any time regardless of whether the animation has completed. The animation should run in the background — if the user scrolls past it before it finishes, that's fine. It continues animating off-screen or stops. It should NEVER hijack scroll behavior or force the viewport position.

Check for any scroll-locking, scroll-snap, or scroll-hijacking JavaScript and remove it. The page should scroll like a normal webpage at all times.
